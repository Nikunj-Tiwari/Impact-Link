# Contributing to ImpactLink

Thank you for your interest in contributing to **ImpactLink**! 🎉

We welcome all contributions — from bug fixes to new features, documentation improvements, and more.

## 📋 How to Contribute

### 1. Fork & Clone

```bash
git clone https://github.com/Nikunj-Tiwari/Impact-Link.git
cd Impact-Link
```

### 2. Create a Branch

Use a descriptive branch name:
```bash
git checkout -b feature/add-new-dashboard-widget
git checkout -b fix/map-clustering-bug
git checkout -b docs/update-setup-guide
```

### 3. Make Your Changes

- Keep changes focused and minimal
- Follow the existing code style
- Add comments where logic is non-obvious

### 4. Commit with Conventional Commits

We use the [Conventional Commits](https://www.conventionalcommits.org/) format:

```
feat: add volunteer location tracking
fix: resolve DBSCAN clustering crash on empty data
docs: update README with new API keys section
chore: upgrade dependencies to latest versions
refactor: extract MapView into standalone component
```

### 5. Push & Open a PR

```bash
git push origin feature/your-feature-name
```

Then open a Pull Request on GitHub targeting the `main` branch.

## 🐛 Reporting Bugs

Open an [Issue](https://github.com/Nikunj-Tiwari/Impact-Link/issues) with:
- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable

## 💡 Suggesting Features

Open a [Discussion](https://github.com/Nikunj-Tiwari/Impact-Link/discussions) or Issue labeled `enhancement`.

## 📜 Code of Conduct

Be respectful, inclusive, and constructive. We're here to build something great together.

---

**Thank you for making ImpactLink better!** 🌐
